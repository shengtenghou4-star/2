# 命镜 · 八字排盘内核

命镜的第一阶段：一个不依赖大模型、结果可导出、边界可核对的八字排盘工作台。

## 当前能力（v0.2）

- 公历出生日期和时间排盘
- 出生地点、经纬度、UTC 标准时差和夏令时显式输入
- 真太阳时修正：夏令时回退、经度修正、均时差分别记账
- 民用时盘与真太阳时盘并列生成、逐柱对照
- 立春定年、节气定月
- 年柱、月柱、日柱、时柱
- 天干十神、地支藏干及藏干十神
- 五行、纳音、十二长生、旬空
- 天干五合与相冲
- 地支六合、六冲、六害、六破、相刑、自刑、三合、三会
- 胎元、胎息、命宫、身宫
- 大运顺逆、起运时间、九步大运和逐年流年
- 晚子时 23:00 换日 / 0:00 换日两种口径
- 双盘、时间修正和结构关系复制为审计 JSON

## 时间校准公式

```text
真太阳时 = 民用钟表时间 − 夏令时 + 4 ×（当地经度 − 时区中央经线）+ 均时差
时区中央经线 = UTC 标准时差 × 15°
```

当前均时差采用分钟级工程近似。历史时区和历史夏令时不由现代时区反推，必须按出生当时制度输入。

## 结构层边界

关系引擎只确认“存在什么结构”，不在本层判断：

- 天干或地支是否真正合化
- 三合、三会是否成局成势
- 刑冲合害的吉凶、强弱和现实应事

这些判断会进入后续旺衰与格局规则层，避免底层排盘越权。

## 架构原则

历法和四柱计算由确定性程序完成。未来的命理规则引擎只读取 `BaziChart` 结构化结果；大模型不参与排盘，也无权修改底层事实。
